# Git Workflow for Bravo FreshFlow AI

This file is the contract for how the repo is committed. Read this before every phase completion.

## At the end of every completed phase

After updating `docs/PROGRESS.md` and BEFORE stopping, do this:

1. Stage all changes:
       git add -A

2. Inspect what will be committed:
       git status

3. Commit with the exact subject-only format (single `-m`):
       git commit -m "<type>: Phase <N> — <short description>"

   Valid types: feat, chore, fix, refactor, docs, style, perf, test

   Examples (use these exact patterns):
   - chore: Phase 0 — project bootstrap and tracking files
   - feat: Phase 2 — app shell and role switcher
   - feat: Phase 3 — executive dashboard
   - feat: Phase 5 — risky products list with filters
   - feat: Phase 6 — product details page
   - feat: Phase 7 — interactive what-if simulator
   - feat: Phase 9 — task management views
   - feat: Phase 14 — polish, animations, demo mode

4. Push:
       git push origin master

5. Verify the author of the last commit:
       git log -1 --pretty=format:"%an <%ae>%n%s"

   It MUST print exactly:
       ismayilzeynal <ismayil.zeynalov.bakhish@bsu.edu.az>
       <type>: Phase <N> — <short description>

   If the author is anything else, STOP and report. Do not push again.

6. Confirm in your final turn summary that push succeeded with the commit hash.

## Strict rules — NEVER violate

- Author MUST be: ismayilzeynal <ismayil.zeynalov.bakhish@bsu.edu.az>
- Single-line subject only. No body. One `-m` flag.
- Forbidden substrings anywhere in any commit message or trailer:
  - "Co-Authored-By: Claude" (or any case variant)
  - "Generated with Claude Code" (or variants)
  - "🤖"
  - "Claude", "AI", "Anthropic", "generated by", "automated by"
  - URLs: claude.com, anthropic.com
- Never amend a pushed commit. Never `--force` or `--force-with-lease`.

## Edge cases

- If `git status` shows no changes, skip the commit and push. Mention "Working tree clean — nothing to commit" in your turn summary.
- If `git push` fails with a merge conflict, STOP and ask the user. Do not attempt to resolve automatically.
- If `git push` fails for auth reasons, STOP and tell the user "Push failed — git authentication needs attention. Please run `git push origin master` manually after verifying credentials."
- If `git log -1` shows a wrong author, STOP and report. Probable cause: global git config overriding local. Tell the user to verify `git config user.name` and `git config user.email` inside this repo.
